Backend Services For iOS Fitness App

About Application

One of the top rankings US based iOS fitness app.

Requirement Gathering

All requirements and conversations generated by them would be put on Trello. The time against each task was also logged there. For lengthy conversation Skype was used.

Project Requirements

Phase 1

Migrate the application backend services, built on Facebook's parse server, to another platform hosting open source parse server. Also, migrate the accompanying mongoDB deployment to a new plan. This included both production and development instances.

Phase 2

Development of:
  • Multiple services to dispatch emails depending on the weekly activity/inactivity of users, new exercise logging etc. Tweak and improve the email sending logic on regular intervals
  • A service that unsubscribes a user from multiple email providers with one click (mandrill and mailchimp)
  • A minimal portal where admin can 1. Switch on/off any of the above background jobs 2. View aggregated data of production db (no of new users, active users; weekly, monthly etc) in graphical representation 3. View and download logs, generated by the above background services and stored in mongoDB, in CSV file
  • Occasional modification/addition in parse server cloud code

    • Implementation

    For phase one the production parse server was deployed on Amazon Elastic Beanstalk. The dev server was hosted on EC2 instance. MongoDBs for both were migrated to an mLab plan.

    In second phase, the backend services were developed with Node, Express and mongoDB (readonly connection to production database and another database inducted for chores). Unit tests written in Mocha. Mandrill the email provider. Angular was used on client side. The deployment was EC2.

    Resource

    Single

    Impact

    The graph of new and active users for the app went many folds up after the integration of activity based weekly emails.

    Year

    2016, 2017

    Duration

    Phase 1

    ~2 weeks

    Phase 2

    Ongoing (I no longer work on it)

    GreenUp

    What's it about?

    Not that I'm perfect, but having seen the condition of spoken and written English of my friends, relatives, colleagues and even higher-ups, I wanted to have some "unique" online presence to impact others in a meaningful way. Pakistan is a third world country where rote learning is the way to success. It's here you'll find doctors, engineers and even PhDs with broken English, although more than half a decade of their higher education was supposedly spent on reading/writing English.

    I've been thinking (over-thinking), assuming the requirements, planning, executing, changing the assumed requirements, planning, executing and repeating the cycle all over. The idea and the application have seen many overhauls.

    When I started, it was a Single Page Angular/Angular material design application with lots of functionalities in mind. Lack of time and urge to speed up forced me to cut down on SPA part and build traditional web application. Further down the line I shunned the idea of application altogether in favor of website. Currently, that's where it stands (though sleeping); a blog/page based informative website.
